## 1. PER (주가수익비율)

### 정의
**Price to Earnings Ratio** — 주가가 주당순이익의 몇 배인지를 나타내는 지표입니다.

### 산출 공식

```
PER = 시가총액 / (TTM 순이익 × 단위배수)
```

| 구성요소 | 설명 |
|---------|------|
| 시가총액 | 현재 주가 × 상장주식수 |
| TTM 순이익 | 최근 4개 분기 순이익 합산 (Trailing Twelve Months) |
| 단위배수 | 삼성전자(005930) 매출 기준으로 추정 (1, 100만, 1억) |

### 특수 처리
- TTM 순이익 ≤ 0인 경우: **NaN** 처리 (적자 기업은 PER 무의미)
- PER < 0.5 또는 > 500인 경우: **PER 이상치 플래그** 활성화 (데이터 품질 경고)

### 해석 기준

| PER 범위 | 해석 |
|----------|------|
| < 5 | 극단적 저평가 또는 이익 급감 우려 |
| 5 ~ 10 | 저평가 영역 |
| 10 ~ 20 | 적정 수준 (시장 평균) |
| 20 ~ 40 | 성장 프리미엄 반영 |
| > 40 | 고평가 또는 고성장 기대 반영 |

### 시스템 내 활용
- **종합점수 밸류에이션 축**: PER Inverse 비중 40% (낮을수록 높은 점수)
- **우량가치 전략**: PEG 계산의 기반
- **스크리닝**: 역수(1/PER) 기준 퍼센타일 랭킹

## 2. PBR (주가순자산비율)

### 정의
**Price to Book Ratio** — 주가가 주당순자산(장부가치)의 몇 배인지를 나타내는 지표입니다.

### 산출 공식

```
PBR = 시가총액 / (자본총계 × 단위배수)
```

| 구성요소 | 설명 |
|---------|------|
| 자본총계 | 최근 재무제표 기준 자기자본 (순자산) |

### 특수 처리
- 자본총계 ≤ 0인 경우: **NaN** 처리 (자본잠식 기업)

### 해석 기준

| PBR 범위 | 해석 |
|----------|------|
| < 0.5 | 극단적 저평가, 자산가치 미만 거래 |
| 0.5 ~ 1.0 | 장부가치 이하, 가치주 영역 |
| 1.0 ~ 2.0 | 적정 수준 |
| 2.0 ~ 5.0 | 무형자산/브랜드 프리미엄 반영 |
| > 5.0 | 고평가, 높은 ROE 기대 반영 |

### 시스템 내 활용
- **종합점수 밸류에이션 축**: PBR Inverse 비중 25%
- **우량가치 전략 Track B (금융주)**: PBR < 1.5 필터
- **S-RIM 산출**: BPS 계산의 기초

---

## 3. PSR (주가매출비율)

### 정의
**Price to Sales Ratio** — 주가가 주당매출의 몇 배인지를 나타내는 지표입니다. 적자 기업에도 적용 가능한 밸류에이션 지표입니다.

### 산출 공식

```
PSR = 시가총액 / (TTM 매출액 × 단위배수)
```

### 특수 처리
- TTM 매출액 ≤ 0인 경우: **NaN** 처리

### 해석 기준

| PSR 범위 | 해석 |
|----------|------|
| < 0.5 | 매우 저평가 (전통 산업) |
| 0.5 ~ 2.0 | 적정 (제조업 기준) |
| 2.0 ~ 10.0 | 성장 프리미엄 또는 고마진 사업 |
| > 10.0 | 고성장 테크/바이오 영역 |

### 활용 포인트
- PER이 NaN인 적자 기업의 밸류에이션 판단에 유용
- 같은 업종 내 비교 시 가장 의미 있음

## 4. PEG (주가수익성장비율)

### 정의
**Price/Earnings to Growth Ratio** — PER을 이익성장률로 나눈 값으로, 성장 대비 밸류에이션을 평가합니다. 피터 린치(Peter Lynch)가 대중화한 지표입니다.

### 산출 공식

```
PEG = PER / min(순이익 CAGR × 100, 200)
```

| 구성요소 | 설명 |
|---------|------|
| PER | 위에서 산출한 값 |
| 순이익 CAGR | 연간 순이익 복합성장률 (최소 2개년) |
| CAGR 캡 | **200%** 상한 제한 (단기 비정상 성장 방지) |

### 해석 기준

| PEG 범위 | 해석 |
|----------|------|
| < 0.5 | 성장 대비 극단적 저평가 |
| 0.5 ~ 1.0 | 성장 대비 저평가 (피터 린치 매수 영역) |
| 1.0 ~ 1.5 | 적정 수준 |
| 1.5 ~ 2.0 | 성장 대비 다소 고평가 |
| > 2.0 | 성장 대비 고평가 |

### 시스템 내 활용
- **우량가치 전략**: PEG < 1.2 하드 필터
- **고성장 전략**: PEG Inverse 비중 20%

## 5. EPS (주당순이익)

### 정의
**Earnings Per Share** — 주식 1주당 벌어들인 순이익입니다.

### 산출 공식

```
EPS = (TTM 순이익 × 단위배수) / 상장주식수
```

### 활용 포인트
- PER 산출의 기반 데이터
- 분기별 EPS 추이로 이익 모멘텀 판단
- DPS와 비교하여 배당성향(Payout Ratio) 산출

---

## 6. BPS (주당순자산)

### 정의
**Book Value Per Share** — 주식 1주당 순자산(장부가치)입니다.

### 산출 공식

```
BPS = (자본총계 × 단위배수) / 상장주식수
```

### 활용 포인트
- PBR 산출의 기반 데이터
- S-RIM 적정가치 산출의 핵심 입력값
- BPS 증가 추세 = 주주가치 축적의 증거

---

## 7. DPS (주당배당금)

### 정의
**Dividend Per Share** — 주식 1주당 지급되는 연간 배당금입니다.

### 데이터 소스
- FnGuide HIGHLIGHT 테이블의 **연간 배당금** 항목
- 분기배당, 추정치는 **제외** (확정 연간 배당만 사용)

### 시스템 내 활용
- 배당수익률(%) = (DPS / 현재가) × 100
- 배당성향(%) = (DPS / EPS) × 100
- DPS CAGR: 배당금 연평균 성장률

---

## 8. S-RIM (초과이익 잔여가치 모델)

### 정의
**Surplus-Residual Income Model** — 사경인 회계사가 한국 시장에 맞게 변형한 잔여이익 모델로, 기업의 초과 수익력을 기반으로 적정 주가를 산출합니다.

### 산출 공식

```
Ke (할인율) = 무위험이자율 + 주식위험프리미엄
적정가치 = BPS + BPS × (ROE - Ke) × ω / (1 + Ke - ω)
괴리율(%) = ((적정가치 - 현재가) / 현재가) × 100
```

| 파라미터 | 기본값 | 설명 |
|---------|--------|------|
| 무위험이자율 | 3.5% | 환경변수 `RISK_FREE_RATE`로 설정 |
| 주식위험프리미엄 | 5.5% | 환경변수 `EQUITY_RISK_PREMIUM`으로 설정 |
| Ke (할인율) | 9.0% | 위 두 값의 합 |
| ω (지속계수) | **동적 0.50~0.95** | 재무품질 기반으로 자동 산출 (하단 참고) |

### 동적 지속계수 ω 산출

ω는 기업의 재무품질에 따라 0.50~0.95 범위에서 동적으로 결정됩니다:

```python
q = (F스코어 / 9)      × 0.30   # 재무건전성
  + (ROIC / 20)         × 0.25   # 자본효율 (0~20% 클립)
  + 영업이익_연속성장    × 0.20   # ≥3년→1.0, ≥1년→0.5, else→0.0
  + (이자보상배율 / 10)  × 0.15   # 0~10배 클립
  + 레버리지_플래그      × 0.10   # 부채<100%→1.0, <200%→0.5, else→0.0

ω = clip(0.50 + q × 0.45, 0.50, 0.95)
```

| 재무품질 수준 | ω 범위 | 적용 대상 예시 |
|-------------|--------|--------------|
| 우량 (q ≈ 1.0) | ~0.95 | 고ROE + 저부채 + 연속성장 기업 |
| 보통 (q ≈ 0.5) | ~0.73 | 일반 상장사 평균 |
| 부실 (q ≈ 0.0) | 0.50 | F스코어 낮고 이자보상배율 위험 |

### ROE 조건별 적정가치 산출

| 조건 | 적정가치 계산 |
|------|-------------|
| ROE > Ke (초과이익) | BPS + BPS × (ROE/100 - Ke) × ω / (1 + Ke - ω) |
| 0 < ROE ≤ Ke (이익但 Ke 미달) | BPS × (ROE / Ke) — 가치 훼손 비율 할인 |
| ROE ≤ 0 (적자) | BPS × 0.5 — 청산가치 50% 적용 |
| BPS ≤ 0 (자본잠식) | NaN |

### 괴리율 해석

| 괴리율 | 해석 |
|--------|------|
| > +50% | 극단적 저평가 (안전마진 충분) |
| +20% ~ +50% | 매력적 저평가 |
| 0% ~ +20% | 소폭 저평가 |
| -20% ~ 0% | 적정 ~ 소폭 고평가 |
| < -20% | 고평가 영역 |

### 시스템 내 활용
- **종합점수 가격 축**: 괴리율 비중 25% (PER역수 35%, PBR역수 20%, FCF 10%, 밸류모델수 10%)
- **우량가치 전략 점수**: 괴리율 비중 20%
- **턴어라운드 전략 점수**: 괴리율 비중 10%

### S-RIM의 한계
- ROE가 일시적으로 높은 경우 적정가치 과대평가
- 자본잠식 기업에는 적용 불가
- 금융업은 자본구조가 달라 해석 주의 필요

## 9. 실전 활용 가이드

### 9.1 밸류에이션 지표 복합 분석 프레임워크

단일 지표만으로 판단하지 말고, **복수 지표를 교차 검증**하는 것이 핵심입니다.

```
[추천 분석 순서]
1단계: PER로 이익 기반 밸류에이션 확인
2단계: PBR로 자산가치 기반 안전마진 확인
3단계: PEG로 성장성 대비 밸류에이션 적정성 확인
4단계: S-RIM 괴리율로 초과이익 기반 적정가치 확인
5단계: FCF Yield로 실제 현금흐름 기반 수익률 확인
```

### 9.2 업종별 적정 밸류에이션 가이드

| 업종 | 주요 지표 | 적정 범위 |
|------|----------|----------|
| 제조업 | PER, PBR | PER 8~15, PBR 0.5~1.5 |
| IT/소프트웨어 | PSR, PEG | PSR 2~8, PEG 0.5~1.5 |
| 금융 | PBR, 배당수익률 | PBR 0.3~0.8, 배당 3%+ |
| 바이오/신약 | PSR | PSR로만 판단 (적자 기업 다수) |
| 유틸리티 | PER, 배당수익률 | PER 8~12, 배당 4%+ |

### 9.3 밸류에이션 트랩 주의사항

**낮은 PER이 항상 좋은 것은 아닙니다:**

| 트랩 유형 | 증상 | 점검 방법 |
|----------|------|----------|
| 실적 피크 | PER 낮지만 이익 감소 전환 | 이익감속 경고(Deceleration Warning) 확인 |
| 일회성 이익 | 비경상 이익으로 PER 왜곡 | TTM vs 연간 이익 비교 |
| 산업 쇠퇴 | 구조적 이익 감소 | 매출 CAGR, 업종 트렌드 확인 |
| 자본잠식 | PBR 매우 낮지만 부실 | 부채비율, 이자보상배율 확인 |

### 9.4 시스템 내 스코어링 방식

본 시스템에서 밸류에이션 지표는 **역수(inverse) 기준 퍼센타일 랭킹**으로 점수화됩니다:

```python
# PER 점수: 낮을수록 높은 점수 (asc=False)
PER_rank = get_rank(df['PER'], asc=False)  # PER 5 → 상위 퍼센타일

# PBR 점수: 낮을수록 높은 점수 (asc=False)
PBR_rank = get_rank(df['PBR'], asc=False)

# 괴리율: 높을수록 높은 점수 (asc=True, 양수 = 저평가)
DIV_rank = get_rank(df['괴리율(%)'], asc=True)
```

### 9.5 종합점수에서의 밸류에이션 가중치

```
종합점수 = 성장(25%) + 안정(35%) + 가격(30%) + 타이밍(15%)

가격 축 (30%) 내부:
├── PER Inverse:  35%
├── 괴리율:      25%
├── PBR Inverse:  20%
├── TTM FCF:     10%
└── 밸류 모델수:  10%
```
