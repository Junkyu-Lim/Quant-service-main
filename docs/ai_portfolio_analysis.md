# AI 포트폴리오 종합 분석 보고서

> **소스 코드**: `analysis/claude_analyzer.py` → `generate_portfolio_report()`

## 개요

투자자의 보유 포트폴리오 전체를 분석하여 포트폴리오 수준의 전략적 조언을 제공합니다. 개별 종목 분석과 달리 **종목 간 관계, 섹터 배분, 리밸런싱 전략**에 초점을 맞춥니다.

- **모델**: `config.PORTFOLIO_MODEL` (기본값: `claude-sonnet-4-6`)
- **max_tokens**: 32,768
- **웹 검색**: 사용하지 않음 (개별 분석과 다름)
- **타임아웃**: 600초 (connect 30초)
- **재시도**: 최대 5회 (429, 529, 500, 502, 503 에러 시 지수 백오프)

---

## 분석 프레임워크 (9단계)

| # | 분석 항목 | 핵심 내용 |
|---|-----------|-----------|
| **1** | 포트폴리오 건강도 평가 | 품질 점수 0-100, 분산도·밸류에이션·성장성·안정성 종합 |
| **2** | **매크로 환경 평가** | 거시경제 환경 요약, 현금 비중 신호, 유리/불리 섹터 (매크로 데이터 제공 시) |
| **3** | 종목별 액션 권고 | BUY_MORE / HOLD / TRIM / SELL + 구체적 매매 수량 (매크로 환경 반영) |
| **4** | 섹터 집중도 분석 | 과집중/과소 섹터, 매크로 우세 섹터 대비 배분 평가 |
| **5** | 리스크 & 촉매 | 상관관계 기반 리스크 + 매크로 리스크(금리·환율·신용), 포트폴리오 전체 촉매 |
| **6** | 적정 종목수 & 구성 최적화 | 현재 종목수 적정성, 전략 유형 판정, 조정 방향 |
| **7** | 배당 분석 | 포트폴리오 배당수익률, 연간 배당 추정, 배당 성향 추세 |
| **8** | 보완 제안 + 관심종목 편입 권고 | 부족한 섹터/테마 + ADD / WATCH / SKIP 판정 |
| **9** | 리밸런싱 실행 계획 | 우선순위 액션 + 실행 시기 권고 |

## 입력 데이터

### 포트폴리오 요약
- 총 종목수, 총평가금액, 총수익률, 섹터 분포

### 종목별 데이터 (`format_portfolio_stock()`)
각 종목마다 아래 정보가 포함됩니다:

- **보유 정보**: 보유수량, 평균매입가, 현재가, 수익률, 비중, 섹터
- **정량 데이터** (`format_quant_data()`): 밸류에이션, 수익성, 성장성, 재무건전성, 배당, 기술적 지표, TTM 실적, 시가총액
- **기존 AI 분석 요약** (있는 경우): 투자 등급, 종합 점수, 핵심 사업, 거장 6인 점수, 리스크/촉매

### 관심종목 (워치리스트)
- 포트폴리오에 없는 관심종목의 정량 데이터 제공
- 편입 여부 검토용

### 상관관계 행렬
- 종목 간 가격 상관관계 (최근 250 거래일 일별 수익률 기준)
- 0.7 이상: 고상관(분산 효과 부족), -0.3 미만: 역상관(분산 효과 양호)

### 매크로 컨텍스트 (선택 제공)
`docs/macro_checklist.md`의 프레임워크에 따라 거시경제 환경을 수동 입력 후 주입합니다. 제공 시 AI가 현금 비중, 섹터 배분, 리밸런싱 판단에 반영합니다. 미제공 시 개별 종목 데이터만으로 분석합니다.

API 요청 body에 `macro_context` 객체를 포함합니다:

## 종목별 액션 권고 상세

### 매매 수량 산출 로직

```
target_shares = abs(총평가금액 × (권장비중 - 현재비중) / 100) ÷ 목표가 중간값
estimated_amount = target_shares × 목표가 중간값
```

| 액션 | target_shares | 가격 범위 | 설명 |
|------|---------------|-----------|------|
| **BUY_MORE** | 추가 매수할 주수 | 매수 목표가 범위 | 비중 확대 |
| **HOLD** | 0 | 현재가 ±5% 감시 범위 | 유지 |
| **TRIM** | 매도할 주수 | 매도 목표가 범위 | 비중 축소 |
| **SELL** | 보유수량 전량 | 매도 목표가 범위 | 전량 매도 |

## 투자 철학 (최우선 전제)
- 1순위: 산업의 장기 성장성(전방산업 CAGR, 해자 등급)
- TRIM/SELL은 산업 성장성 훼손, 해자 소실, 구조적 경쟁력 약화 확인 시에만 권고
- 단기 고평가(PER 등)는 TRIM 근거가 되지 않음

## 분석 프레임워크
1. 포트폴리오 건강도 평가 (0-100)
2. 종목별 액션 권고 (BUY_MORE/HOLD/TRIM/SELL + 구체적 매매 수량)
3. 섹터/산업 성장성 분석
4. 포트폴리오 리스크 & 촉매
5. 적정 종목수 & 구성 최적화
6. 배당 분석
7. 보완 제안
8. 관심종목 편입 권고 (ADD/WATCH/SKIP)
9. 리밸런싱 실행 계획

## 매크로 환경 활용 지침 (매크로 컨텍스트 제공 시)
- 현금 비중: 방어적 환경(성장 하향 + 실질금리 상승 + 금융여건 악화)이면 현금 유지/확대 권고.
  공격적 환경이면 예수금 범위 내 BUY_MORE 적극 검토.
- 섹터 배분: 매크로 유리 섹터 종목에 BUY_MORE 가중, 불리 섹터 과집중 시 TRIM 검토.
  단, 장기 성장성과 해자가 충분하면 매크로 불리라도 HOLD 우선.
- 리스크: 매크로 리스크(금리 상승, 달러 강세, 신용 악화)를 portfolio_risks에 포함.
- 매크로 컨텍스트가 없으면 이 지침을 무시하고 개별 데이터만으로 분석.

## 분석 지침
- 포트폴리오 비중이 높은 종목에 더 주의를 기울이세요
- 종목 간 상관관계와 중복 리스크를 식별하세요
- 데이터가 없는 종목(ETF, 우선주 등)은 비중 분석에만 포함
- 6대 투자 거장 관점이 기존 분석에 포함되어 있으면 종합 활용
- 한국어로 분석

## 캐시 & 무효화

- 분석 결과는 `portfolio_analysis` 테이블에 1건만 저장
- 포트폴리오 구성(종목코드, 수량, 평균매입가)의 MD5 해시로 캐시 유효성 판단
- 포트폴리오 변경 시 `stale=true` 반환 → UI에 "포트폴리오가 변경되었습니다" 배너 표시

## 개별 분석과의 차이점

| 항목 | 개별 종목 분석 | 포트폴리오 분석 |
|------|----------------|-----------------|
| 모델 | `ANALYSIS_MODEL` (sonnet) | `PORTFOLIO_MODEL` (sonnet) |
| 웹 검색 | 사용 (최대 2회) | 미사용 |
| max_tokens | 16,384 | 32,768 |
| 분석 단위 | 종목 1개 | 포트폴리오 전체 |
| 핵심 관점 | 거장 6인 프레임워크 | 배분·상관관계·리밸런싱 |
| 입력 | 정량 데이터만 | 정량 + 기존 AI 분석 + 상관관계 + 관심종목 + 매크로 컨텍스트(선택) |
| 매크로 반영 | 없음 | 현금 비중·섹터 배분·리스크에 반영 (데이터 제공 시) |
| 캐시 키 | 종목코드 | 포트폴리오 구성 해시 |

## 포트폴리오 관리 기능 (신규)

포트폴리오 탭은 5개의 서브탭으로 구성됩니다:

### 1. 보유종목 탭 (기존)
- 보유 종목 테이블 (수익률, 비중, 퀀트점수 등)
- 종목 추가/수정/삭제
- AI 포트폴리오 분석 버튼

### 2. 수익률 추이 탭
포트폴리오 전체 에퀴티 커브를 차트로 시각화합니다.

- **데이터 소스**: `price_history` 테이블 (최대 260 거래일)
- **기간 선택**: 1M / 3M / 6M / 1Y
- **라인**: 포트폴리오 합산 수익률 (파란 굵은선) + KOSPI 벤치마크 (회색 점선) + 개별 종목 (범례 클릭으로 토글)
- **API**: `GET /api/portfolio/performance?range=3M`
- **주의**: price_history에 없는 종목(ETF 등)은 차트에서 제외될 수 있음

### 3. 건강 상태 탭
각 보유 종목의 퀀트 지표와 AI 분석 결과를 종합하여 건강도를 표시합니다.

**퀀트 지표** (dashboard_result 기반):
| 지표 | 초록 | 노랑 | 빨강 |
|------|------|------|------|
| 수익률 | ≥ 0% | -10%~0% | < -10% |
| RS등급 | ≥ 70 | 40~69 | < 40 |
| 퀀트점수 | ≥ 70 | 40~69 | < 40 |
| 52주고가 대비 | ≥ -10% | -10%~-25% | < -25% |
| F스코어 | ≥ 6 | 4~5 | < 4 |

**AI 지표** (analysis_reports 기반, 개별 종목 AI 분석 시 생성):
| 지표 | 초록 | 노랑 | 빨강 |
|------|------|------|------|
| AI 거장 평균 | ≥ 7/10 | 5~6.9 | < 5 |
| 해자등급 | Wide/Narrow | - | None |

- 빨간 지표 2개 이상인 종목은 경고 배너에 표시
- AI 분석 미실시 종목은 AI 컬럼 "N/A" 표시, 경고 판단 제외
- 포트폴리오 AI 분석 보고서 freshness 표시 (7일 초과 시 "재분석 권장")
- **API**: `GET /api/portfolio/health`

### 4. 거래 기록 탭
종목 추가·수정·삭제 시 자동으로 기록됩니다.

- **자동 기록**: 포트폴리오 CRUD 작업 시 `portfolio_transactions` 테이블에 저장
- **거래 유형**: 매수(BUY) / 매도(SELL) / 조정(ADJUST)
- **표시 정보**: 날짜, 종목명, 유형, 수량, 단가, 거래금액, 메모
- **API**: `GET /api/portfolio/transactions?limit=200`

> **참고**: 기존에 등록된 종목들은 거래 기록이 없습니다. 이후 추가·수정·삭제부터 기록됩니다.

### 5. 리밸런싱 탭
현재 보유 비중과 목표 비중의 차이를 계산하여 조정 가이드를 제공합니다.

- **기본 모드**: 동일비중 (100% ÷ 보유 종목 수)
- **커스텀 모드**: 목표비중 직접 입력 후 "목표 저장" → `portfolio_targets` 테이블에 저장
- **조정 기준**: 편차 ±2% 이상인 종목만 조정 제안
- **조정량 계산**: `편차% × 총평가금액 ÷ 현재가`
- **시각화**: 종목별 현재비중(파란 바) + 목표비중(빨간 세로선)
- **API**: `GET /api/portfolio/rebalance` / `POST /api/portfolio/rebalance/targets`

## 포트폴리오 관리 활용 워크플로우

```
1. 종목 추가/매수 → 포트폴리오 탭에서 "+ 종목 추가"
   → 자동으로 거래 기록에 "매수" 기록

2. 정기 점검 (주 1회 권장)
   a. "건강 상태" 탭 확인 → 경고 종목 식별
   b. "수익률 추이" 탭으로 포트폴리오 vs KOSPI 성과 비교
   c. "AI 포트폴리오 분석" 실행 → 종목별 액션(BUY_MORE/HOLD/TRIM/SELL) 확인

3. 매크로 점검 (월 1회 권장, macro_checklist.md 참조)
   a. 성장 방향 / 실질금리 / 금융여건 → 현금 비중 판단 (공격적/중립/방어적)
   b. 환율 / 원자재 / CAPEX 테마 → 섹터 배분 방향 설정
   c. 판단 결과를 macro_context JSON으로 정리하여 "AI 포트폴리오 분석" 요청에 포함
   → AI가 매크로 환경을 반영한 현금 비중 및 섹터 배분 권고 생성

4. 리밸런싱 검토 (월 1회 권장)
   a. "리밸런싱" 탭 → 동일비중 또는 목표비중 설정
   b. 편차가 큰 종목 조정
   c. 조정 후 포트폴리오 수정 → 거래 기록 자동 업데이트

5. 종목 교체 판단 기준
   - 수익률 < -10% + RS등급 < 40 → 모멘텀 이탈 신호
   - 종합점수 < 40 + F스코어 < 4 → 펀더멘털 악화 신호
   - AI 거장 평균 < 5 → 정성적 매력도 저하
   - 매크로 불리 섹터 + 해자 Narrow/None → TRIM 후보 우선 검토
```
